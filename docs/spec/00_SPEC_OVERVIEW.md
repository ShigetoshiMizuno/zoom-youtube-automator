## 1. プロジェクト概要

### 目的

調布南キリスト教会の礼拝配信業務を自動化する。礼拝情報を入力し「開始」ボタンを押すだけで、OBS録画・Zoom配信・YouTube自動アップロードを一括完結させるデスクトップアプリ `zoom-youtube-automator` を開発する。

### 背景

礼拝ごとに以下の手作業が発生しており、担当者の技術的負担が大きい。

- OBSでの録画開始・停止操作
- Zoomミーティングの手動起動・参加
- 録画後のYouTubeアップロード（タイトル・説明文・サムネイルの手入力）
- サムネイル画像の手動作成

2026-04-20の礼拝では人員不足により音声のみ手動収録・手動変換でYouTubeにアップロードした。この反省を踏まえ、非技術者でも1ボタンで完結できる仕組みを整備する。

### 対象ユーザー

- 主担当: 調布南キリスト教会のスタッフ（IT非専門家を含む）
- 使用環境: Windows PC（礼拝会場の配信用PC）
- 前提スキル: アプリ起動・テキスト入力・ボタン操作ができれば十分

## 2. 確定要件一覧

### 2.1 機能要件

#### 録画・音声収録

- 映像録画は必須（カメラ映像 + マイク／ライン音声の同期収録）
- OBS WebSocket API（v5）でOBSを制御して録画を開始・停止する
- 録画形式: MP4（YouTube直接アップロード可能な形式）
- 録画保存先: `config.yaml` の `obs.output_dir` で設定
- 固定カメラ1台構成（将来的な複数カメラ対応はFuture Scope）

#### Zoom配信

- `zoommtg://` URLスキームでZoomクライアントを自動起動・ミーティング参加
- pywin32でZoomウィンドウの前面表示・配置制御を行う
- OBS仮想カメラ機能でOBSの映像をZoomのカメラ映像として提供する
- 毎回同じミーティングID（`config.yaml` の `zoom.meeting_id` で固定）
- Zoom側のRTMP/カスタムライブストリーミング設定は不要

#### サムネイル自動生成

- 礼拝情報入力（日付・タイトル・聖書箇所・説教者）からサムネイル画像を自動生成
- 生成ライブラリ: Pillow
- デザインテンプレート: 調布南キリスト教会デザイン（実績あり）
- フォント: Windows Meiryo（`C:/Windows/Fonts/meiryob.ttc`）
- 出力ファイル: `thumbnail.png`（録画保存先と同一フォルダ）

#### YouTube自動アップロード

- YouTube Data API v3 を使用
- 録画終了後に自動でアップロード開始
- タイトル形式: `{年}年{月}月{日}日 「{タイトル}」 {聖書箇所} {説教者}師`
  - 例: `2026年4月19日 「忘れ得じ、彼のパリサイびと」 ヨハネの福音書3章1〜15節 前田重雄師`
- 説明文: TBD（テンプレートをconfig.yamlで定義予定）
- サムネイル: 自動生成した `thumbnail.png` を自動セット
- 公開設定: UIで「公開」「限定公開」を選択可能（デフォルト: 公開）
- 認証: OAuth2（固定Googleアカウント、リフレッシュトークンをconfig.yamlに保存）

### 2.2 非機能要件

| 項目 | 要件 |
|------|------|
| 対応OS | Windows 10 / 11（64bit） |
| 軽量動作 | GUIは Tkinter（追加インストール不要）。Python標準ライブラリを優先 |
| OBSバージョン | v28 以上（WebSocket v5 標準搭載バージョン） |
| 起動速度 | アプリ起動から操作可能まで5秒以内 |
| 設定ファイル | config.yaml（gitignore対象、secrets管理） |
| 日本語対応 | UI表示・ファイル名・ログはすべて日本語対応 |
| エラー表示 | 通信エラー・API認証エラーはGUI上にわかりやすいメッセージで表示 |

### 3.1 コンポーネント構成図

```
+--------------------------------------------------+
|              zoom-youtube-automator              |
|                  (Python / Tkinter)              |

|            v                   v          v      |
|  +----------+  +------------+  +----------+      |
|  | OBS連携  |  | Zoom連携   |  |YouTube連携|      |
|  |(SPEC_OBS)|  |(SPEC_ZOOM) |  |(SPEC_YT) |      |
|  +----+-----+  +------------+  +----------+      |
|       |                                          |

        v               v                v
  +----------+  +---------------+  +------------+
  |  OBS     |  | Zoom Client   |  | YouTube    |
  |(localhost |  |(zoommtg://)   |  | Data API   |
  | :4455)   |  |+ pywin32      |  | v3         |
  +----------+  +---------------+  +------------+
```

### 3.2 データフロー

#### 「開始」ボタン押下時

```
GUI入力値（日付・タイトル等）
    |
    v
コントローラー
    +-> OBS連携: WebSocket接続確認 -> 録画開始リクエスト
    +-> Zoom連携: zoommtg://URL生成 -> Zoomクライアント起動 -> pywin32でウィンドウ制御

    ※ OBS仮想カメラは事前に有効化されていることを前提とする（TBD: 自動有効化の可否）
```

#### 「終了 → YouTubeへアップロード」ボタン押下時

```
コントローラー
    +-> OBS連携: 録画停止リクエスト -> 録画ファイルパス取得
    +-> サムネイル生成: GUI入力値 -> Pillow -> thumbnail.png 生成
    +-> YouTube連携: OAuth2認証確認 -> 動画アップロード -> サムネイルセット -> 公開設定

    ※ Zoomの終了は手動（ユーザーがZoomクライアントを閉じる）
```

## 5. ユーザーストーリー & 操作フロー

### 5.1 通常フロー（礼拝当日）

```
[事前準備]
1. OBSを起動し、仮想カメラを有効にしておく（手動またはOBS起動スクリプト）
2. zoom-youtube-automator を起動する
   -> アプリ起動時にOBS WebSocket接続確認を自動実施する
   -> OBSに接続できない場合は警告メッセージを表示する（起動はブロックしない）
   -> 仮想カメラが未有効の場合も警告メッセージを表示し、有効化を促す

[礼拝情報入力]
3. 日付・タイトル・聖書箇所・説教者を入力する
4. 公開設定を選択する（デフォルト: 公開）

[配信開始]
5. 「▶ 配信・録画 開始」ボタンを押す
   -> OBS録画を自動開始
   -> Zoomを自動起動・ミーティング参加
   -> 状態表示が「配信中」に変わる

[礼拝実施]
6. 礼拝を進行する（アプリ操作は不要）

[終了・アップロード]
7. 「■ 終了 → YouTubeへアップロード」ボタンを押す
   -> OBS録画停止
   -> サムネイル自動生成
   -> YouTube自動アップロード（タイトル・説明文・サムネイル付き）
   -> 状態表示が「アップロード中」→「完了」に変わる

8. Zoomクライアントを手動で終了する（TBD: 自動終了の可否を検討）
```

### 5.2 エラー発生時フロー

```
- OBS未起動 / 接続失敗 : GUI上にエラーメッセージ表示、処理中断
- Zoom起動失敗         : GUI上にエラーメッセージ表示（録画は継続）
- YouTube認証切れ      : 再認証ダイアログを表示
- YouTubeアップロード失敗: エラー詳細をGUI表示、手動リトライボタン提供（TBD）
```

## 8. 設定ファイル仕様（config.yaml）

ファイルは `.gitignore` の対象。テンプレートとして `config.yaml.example` をリポジトリに含める。

```yaml
obs:
  host: localhost          # OBS WebSocket サーバーのホスト名（通常 localhost）
  port: 4455               # OBS WebSocket サーバーのポート（デフォルト: 4455）
  password: "xxxxx"        # OBS WebSocket 認証パスワード
  scene: "礼拝"            # 録画に使用するOBSシーン名（要ユーザー確認）
  output_dir: "C:/録画"    # 録画ファイルの保存先ディレクトリ

zoom:
  meeting_id: "123-456-7890"     # 毎回使用する固定ミーティングID
  password: "xxxxx"              # Zoomミーティングのパスワード

youtube:
  account: "your@gmail.com"      # アップロード先Googleアカウント（表示用）
  default_visibility: "public"   # デフォルト公開設定: "public" or "unlisted"
  credentials_path: "client_secret.json"  # OAuth2クライアントシークレットファイルのパス
  token_path: "token.json"                # OAuth2トークンキャッシュファイルのパス（認証後に自動生成）
  description_template: ""               # YouTubeアップロード時の説明文テンプレート
```

セクション別キー説明：

| セクション | キー | 型 | 説明 |
|-----------|------|----|---------|
| obs | host | string | OBS WebSocket サーバーアドレス。通常は `localhost` |
| obs | port | integer | OBS WebSocket ポート番号。OBSデフォルト: 4455 |
| obs | password | string | OBS WebSocket 認証パスワード。OBS設定で確認 |
| obs | scene | string | 録画対象シーン名（TBD: ユーザーに確認要） |
| obs | output_dir | string | 録画ファイル保存ディレクトリの絶対パス |
| zoom | meeting_id | string | 固定ミーティングID（ハイフン区切り形式） |
| zoom | password | string | Zoomミーティングのパスコード |
| youtube | account | string | アップロード先Googleアカウントのメールアドレス（表示用） |
| youtube | default_visibility | string | `"public"` または `"unlisted"` |
| youtube | credentials_path | string | OAuth2クライアントシークレットJSONファイルのパス（Google Cloud Consoleからダウンロード） |
| youtube | token_path | string | OAuth2トークンキャッシュファイルのパス。初回認証後に自動生成される `token.json` ファイル |

> **セキュリティ注意:** `credentials_path` に指定するクライアントシークレットおよび `token_path` のトークンキャッシュファイルはいずれも機密情報。`config.yaml` とともに `.gitignore` に含め、ファイルへのアクセス権限を適切に設定すること。

## 10. リスク・制約事項

### 10.1 OBSバージョン要件

- WebSocket API v5 は OBS v28 以降に標準搭載
- v28 未満の環境では `obs-websocket` プラグインを別途インストールする必要がある
- **現状:** 実際のOBSバージョンは未確認。実機で `OBS > ヘルプ > バージョン情報` を確認すること

### 10.2 Zoomバージョン変更リスク

- `zoommtg://` URLスキームの仕様はZoom公式には明文化されておらず、バージョンアップで挙動が変わる可能性がある
- **軽減策:** 自動起動が失敗した場合のフォールバック（エラーメッセージ表示 + 手動起動の案内）を実装する

### 10.3 OAuth2トークン管理

- YouTube OAuth2トークンは `token.json` ファイルで管理する（`config.yaml` への直接保存は廃止）
- `credentials_path`（credentials.json）と `token_path`（token.json）を config.yaml で指定する
- Googleアカウントの認証情報が漏洩するリスクがある
- **軽減策:** `token.json` / `credentials.json` を gitignore 対象とし、ファイルへのアクセス権限を適切に設定すること
- **将来検討:** Windows資格情報マネージャー（keyring）への保存も選択肢として残す

### 10.4 Windows CP932エンコーディング問題

- 日本語ファイル名・パスを含む操作では `subprocess` 経由の実行でエラーが発生することがある（実際に発生済み）
- **確定回避策:** ファイル操作は `subprocess` でなく Python の `os` / `glob` / `shutil` を使用する
- **ffmpegへの日本語パス渡し:** 直接渡さず、一時ファイル（ASCII名）にコピーしてから渡す
- **リネーム時のPermissionError:** ffmpeg完了直後はファイルがロックされることがある。`shutil.move` の前にプロセス完了確認を行う

### 10.5 OBS仮想カメラの事前有効化

- OBS仮想カメラはOBS WebSocket APIから制御できない場合がある（バージョン依存）
- **現状の前提:** ユーザーがOBS起動時に手動で仮想カメラを有効にする
- **TBD:** アプリからの自動有効化の可否を調査・検討

### 10.6 動作確認済み環境

| 機能 | 状態 |
|------|------|
| サムネイル生成（Pillow + Meiryo） | 動作確認済み |
| ffmpegによるm4a→MP4変換 | 動作確認済み |
| OBS WebSocket連携 | 未実装・未確認 |
| Zoom連携 | 未実装・未確認 |
| YouTube API連携 | 未実装・未確認 |
| GUIアプリ | 未実装 |

### 10.7 機能優先順位・運用前提

- **録画が最優先機能**であり、OBS録画の開始・停止が確実に機能することを絶対条件とする
- Zoom自動起動・YouTube自動アップロードはベストエフォートとして扱う（失敗してもアプリをクラッシュさせない）
- Zoom起動失敗時は録画を継続し、GUI上にエラーを表示する
- YouTubeアップロード失敗時は録画ファイルを保持し、手動アップロードで救済可能な状態を維持する
- **専用機前提:** 本アプリは礼拝配信専用PCで動作することを前提とする。他アプリとの競合やリソース不足は想定しない

### 10.8 録画ファイル書き込み完了待ち

- OBS録画停止リクエスト送信後、MP4ファイルへの書き込みが完了するまでに数秒〜十数秒かかる
- 書き込み完了前にアップロードを開始すると不完全なファイルがYouTubeに送信されるリスクがある
- **必須対処:** 録画停止後、ファイルサイズが一定時間変化しなくなるまで待機してからアップロードを開始する
- **具体的実装方針:** OBS WebSocketの `RecordingStopped` イベント受信後に、ファイルサイズポーリング（例: 1秒間隔×10回チェックで安定確認）を行ってからアップロード処理へ進む
