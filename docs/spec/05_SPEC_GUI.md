## 概要

Tkinter を使用したデスクトップ GUI アプリ本体の仕様。礼拝情報の入力フォーム、配信開始／終了ボタン、状態表示を提供し、OBS・ Zoom・サムネイル・ YouTube の各モジュールをオーケストレーションする。

本モジュール（app.py）は GUI の描画・イベントハンドリング・状態管理を担う。各外部モジュールの呼び出しはコントローラー層（後述のインターフェース定義）を経由して行う。

## 機能仕様

- 起動時に礼拝情報入力フォームを表示する
- 日付フィールドは当日の日付を初期値として自動設定する
- 起動時に OBS 接続確認を行い、画面に「OBS: ✓接続済み / ✗未接続」インジケーターを表示する
- 「開始」ボタン押下で OBS 録画開始・ Zoom 自動起動を実行する
- 「終了 → YouTubeへアップロード」ボタン押下時は「礼拝を終了してYouTubeにアップロードします。よろしいですか？」確認ダイアログを表示し、OK 時のみ処理を続行する
- 「終了 → YouTubeへアップロード」ボタン押下で OBS 録画停止・サムネイル生成・ YouTube アップロードを実行する
- 各処理はバックグラウンドスレッドで実行し、メインスレッド（UI）をブロックしない
- 処理状態はラベルとプログレスバーでリアルタイムに表示する
- 配信中は経過時間を「配信中 (HH:MM:SS)」形式で 1 秒ごとに更新して表示する
- エラー発生時はダイアログおよび状態ラベルでユーザーに通知する（エラーメッセージは非技術者向けの日本語で「次にすべき行動」を含む形式とする）
- ボタンは状態遷移に応じて有効／無効を切り替える
- 録画中・アップロード中にウィンドウを閉じようとした場合は確認ダイアログを表示し、終了確認時は OBS に停止命令を送ってから終了する
- アップロード完了後に YouTube URL を状態ラベルおよびダイアログに表示し、クリックでブラウザが開く
- アップロード失敗時は録画ファイルの保存先パスをダイアログに表示し「エクスプローラーで開く」ボタンを提供する
- 30 秒ごとに OBS の録画状態をポーリングし、意図せず停止していた場合はユーザーに警告する
- アップロード完了後、「次の礼拝の準備をする」ボタンでタイトル・聖書箇所・説教者フィールドをクリア（日付は当日で再設定）する

## ボタン仕様

### 「開始」ボタン（btn_start）

| 項目 | 仕様 |
|------|------|
| 表示テキスト | ▶  配信・録画 開始 |
| 背景色（有効時） | #28a745（緑）|
| 文字色（有効時） | 白 |
| 背景色（無効時） | #cccccc（グレー）|
| 押下時の処理 | 後述「開始処理フロー」を参照 |

#### 開始処理フロー（on_start_click）

1. 入力バリデーション（日付・タイトル・聖書箇所・説教者が空でないか確認）
   - いずれかが空の場合: messagebox.showwarning で警告ダイアログを表示して中断
2. ボタン状態を「録画中」に遷移（後述の状態遷移表を参照）
3. 録画開始時刻を記録し、経過時間タイマーを開始する（1 秒ごとに root.after で lbl_status を更新）
4. バックグラウンドスレッドで以下を並行実行:
   - obs_client.start_recording() を呼び出す
   - zoom_module.launch_meeting() を呼び出す
5. OBS 接続失敗時: 非技術者向けエラーメッセージを messagebox.showerror で表示、状態を「エラー」に遷移
6. Zoom 起動失敗時: 非技術者向け警告メッセージを messagebox.showwarning で表示（OBS 録画は継続）

### 「終了 → YouTubeへアップロード」ボタン（btn_stop）

| 項目 | 仕様 |
|------|------|
| 表示テキスト | ■  終了 → YouTubeへアップロード |
| 背景色（有効時） | #dc3545（赤）|
| 文字色（有効時） | 白 |
| 背景色（無効時） | #cccccc（グレー）|
| 押下時の処理 | 後述「終了処理フロー」を参照 |

#### 終了処理フロー（on_stop_click）

1. messagebox.askokcancel で「礼拝を終了してYouTubeにアップロードします。よろしいですか？」確認ダイアログを表示する
   - キャンセル時: 何もせず処理を中断する
2. 経過時間タイマーを停止する
3. ボタン状態を「アップロード中」に遷移
4. バックグラウンドスレッドで以下をシーケンシャルに実行:
   1. obs_client.stop_recording() を呼び出す → 録画ファイルパスを取得
   2. thumbnail_module.generate(form_data) を呼び出す → thumbnail.png 生成
   3. youtube_module.upload(video_path, metadata, thumbnail_path, progress_callback) を呼び出す
      - 進捗コールバックで progress_bar を更新する
5. アップロード成功: 状態を「完了」に遷移し、YouTube URL を表示する（後述）
6. アップロード失敗: 録画ファイルパスを含む非技術者向けエラーダイアログを表示し、状態を「エラー」に遷移

### ボタン状態遷移表

| 状態 | btn_start | btn_stop | 入力フォーム |
|------|-----------|----------|------------|
| 待機中（起動直後） | 有効 | 無効 | 有効 |
| 録画中 | 無効 | 有効 | 無効 |
| アップロード中 | 無効 | 無効 | 無効 |
| 完了 | 有効（リセット可） | 無効 | 有効 |
| エラー | 有効（リセット可） | 無効 | 有効 |

| 項目 | 仕様 |

| ウィジェット | tk.Label |

| 表示位置 | メインウィンドウ最上部（入力フォームの上）|
| 確認タイミング | アプリ起動時（非同期）|

### 表示状態

| 状態 | 表示テキスト | 文字色 |
|------|-------------|--------|
| 接続確認中 | OBS: 確認中... | #ffa500（オレンジ）|
| 接続済み | OBS: ✓接続済み | #28a745（緑）|
| 未接続 | OBS: ✗未接続 | #dc3545（赤）|
| 仮想カメラ無効 | OBS: ✓接続済み（仮想カメラが無効です） | #ffa500（オレンジ）|

### 接続確認の動作

- 接続成功かつ仮想カメラ有効: lbl_obs_status を「✓接続済み」（緑）に設定する
- 接続成功かつ仮想カメラ無効: lbl_obs_status を「✓接続済み（仮想カメラが無効です）」（オレンジ）に設定し、messagebox.showwarning で「OBSの仮想カメラが有効になっていません。OBSで仮想カメラを起動してから配信を開始してください。」と警告する

## 経過時間表示

録画中（「配信中」状態）にのみ経過時間を表示する。

- 録画開始時刻を _recording_start_time: datetime.datetime として記録する
- root.after(1000, _update_elapsed_time) による再帰的コールバックで 1 秒ごとに更新する
- 表示形式: 状態: 配信中 (HH:MM:SS)（例: 状態: 配信中 (00:23:15)）
- 「終了」ボタン押下後はタイマーを停止し、以降のコールバックをキャンセルする（_elapsed_timer_id を保持し root.after_cancel で停止）

---

## OBS 録画状態ポーリング

録画中（「配信中」状態）において、30 秒ごとに OBS の録画状態を確認する。

- root.after(30000, _poll_obs_status) による再帰的コールバックで実装する
- _poll_obs_status は obs_client.get_recording_status() を呼び出す
- 録画が継続中: 何もしない（次の 30 秒後のポーリングをスケジュールする）
- 録画が意図せず停止していた場合:
  - lbl_status のテキストを「⚠ 録画が停止しています」、色を #dc3545（赤）に変更する
  - messagebox.showwarning で「録画が停止しています。OBSを確認し、必要に応じて手動で録画を再開してください。」を表示する
  - ポーリングを停止する（継続しない）
- 「終了」ボタン押下後または録画停止後はポーリングを停止する

## エラーメッセージ仕様

### 基本方針

エラーメッセージは非技術者（教会スタッフ）が見ることを前提とし、「次にすべき行動」を含む日本語で記述する。システム例外メッセージ（str(e) 等）は画面に表示しない。技術的な詳細はログファイルにのみ記録する。

### エラー種別ごとのメッセージ定義

| エラー種別 | 例外クラス | 表示メッセージ |
|-----------|-----------|--------------|
| OBS 接続失敗 | OBSConnectionError | OBSが起動していません。OBSを起動して仮想カメラを有効にしてから、もう一度「開始」を押してください。 |
| Zoom 起動失敗 | ZoomLaunchError | Zoomの起動に失敗しました。Zoomがインストールされているか確認してください。（録画は継続しています） |
| YouTube 認証エラー | YouTubeAuthError | YouTube認証が必要です。ブラウザが開くのでGoogleアカウントにログインしてください。 |
| アップロード失敗 | YouTubeUploadError | アップロードに失敗しました。録画ファイルは {path} に保存されています。 |
| 録画ファイル未検出 | OBSError（ファイル未検出） | 録画ファイルが見つかりませんでした。OBSの録画保存先フォルダを確認してください。 |
| 想定外のエラー | Exception | 予期しないエラーが発生しました。アプリを再起動して試してください。（詳細はログファイルを確認してください） |

### ログ記録

logging モジュールを使用し、すべての例外を DEBUG/ERROR レベルでログファイルに記録する。ログファイルのパスは config.yaml の log_path で指定する（未設定時は app.log をカレントディレクトリに作成）。

## アップロード完了後の YouTube URL 表示

youtube_module.upload() の戻り値として YouTube 動画 URL（https://www.youtube.com/watch?v=XXXXXXXXXXX 形式）を受け取り、以下の方法でユーザーに提示する。

1. 状態ラベル: 状態: アップロード完了 ▶ https://youtu.be/XXXX を表示する。テキストをクリックすると webbrowser.open(url) でブラウザが開く（tk.Label に <Button-1> バインド）
2. 完了ダイアログ: messagebox.showinfo でアップロード完了を通知し、YouTube URL を表示する

URL のクリック動作は lbl_status に cursor="hand2" を設定し、クリック時に webbrowser.open(youtube_url) を呼び出す。

## ウィンドウ閉じ操作のハンドリング

root.protocol("WM_DELETE_WINDOW", on_close_handler) を設定し、ウィンドウの × ボタンが押されたときに on_close_handler を呼び出す。

### on_close_handler の動作

| アプリ状態 | 動作 |
|----------|------|
| 待機中・完了・エラー | 確認なしにアプリを終了する（root.destroy()）|
| 録画中 | messagebox.askokcancel で「録画中です。本当に終了しますか？」を表示する。OK 時は obs_client.stop_recording() を呼び出してから root.destroy() する |
| アップロード中 | messagebox.askokcancel で「アップロード中です。本当に終了しますか？」を表示する。OK 時はそのまま root.destroy() する（アップロードはキャンセルされる）|

録画中に OBS 停止命令を送る際は例外を無視して終了処理を続行する（二重エラーを防ぐため）。

## プログレスバー

| 項目 | 仕様 |
|------|------|
| ウィジェット | ttk.Progressbar |
| 変数名 | progress_bar |
| モード | determinate（0〜100 の進捗値を表示）|
| 表示タイミング | YouTube アップロード開始時に表示（pack または grid）|
| 非表示タイミング | アップロード完了・エラー・待機中はウィジェットを非表示（pack_forget）|
| 値の更新方法 | YouTube アップロードの進捗コールバックから progress_bar["value"] = n をメインスレッドに伝達する（root.after を使用）|

### 進捗値の計算根拠

YouTube Data API v3 の MediaFileUpload はチャンクアップロードをサポートする。チャンク送信ごとに送信済みバイト数を取得し、総ファイルサイズに対する割合（0〜100）を progress_bar に反映する。

## エラーダイアログ一覧

tkinter.messagebox を使用する（アップロード失敗時のみカスタムダイアログを使用する）。

| 場面 | 関数 | メッセージ |
